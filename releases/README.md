# Releases

All production release notes live in this folder.

## Naming Convention

- One file per released version.
- Use the exact filename format: `vX.Y.Z.md`.

Examples:

- `v1.0.0.md`
- `v1.0.1.md`

## Required Fields

Each release file should include:

- Version
- Deployed commit
- Deployment date (UTC)
- Change summary
